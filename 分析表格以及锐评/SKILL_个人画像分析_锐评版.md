# B站用户评论个人画像分析（锐评版）

从 B站用户公开评论中，像侦探一样推断个人信息，像毒舌评论员一样写分析报告。

## 触发条件

当用户提到以下关键词时使用此 skill：
- "分析这个B站用户的评论"
- "根据评论推断个人信息"
- "锐评" + "评论"
- "B站用户画像分析"

## 前置数据

需要先有评论 CSV 文件（由爬虫脚本生成），包含字段：
| 字段 | 说明 |
|------|------|
| 序号 | 1-based 行号 |
| 评论ID | B站 rpid |
| 评论内容 | 评论文本 |
| 发布时间 | `YYYY-MM-DD HH:MM:SS` |
| 时间戳 | Unix timestamp (秒) |
| 评论层级 | 顶级评论 / 子评论 |
| 动态ID | B站 oid |
| 动态类型 | 视频 / 专栏 / 动态 |
| 视频标题 | 从 B站 API 获取 |
| 跳转链接 | B站评论直链 |

## 分析流程

### 第一步：读取 CSV

从 `output/` 目录读取最新评论 CSV，确认总条数和时间跨度。

### 第二步：逐条阅读评论

通读全部评论，对每条评论标记：
- 是否包含个人信息（地点、年龄、性别等）
- 是否暴露兴趣爱好
- 是否展现性格特征
- 语气和态度

### 第三步：逐维度推断

按以下类别组织分析：

#### A. 基本信息
- 性别：查找自称「我女/男的」「我一女的」等表述
- 年龄段：从「学校」「班级」「上课」「暑假」等关键词推断
- 所在地：从「来XX看看」「XX的天气」等地理描述推断
- 手机型号：从「我的XX手机」等设备吐槽推断
- 身高体重：从自曝数据推断
- 家庭关系：从提到父母的评论推断

#### B. 职业与学业
- 职业身份：学生/已工作（「我们学校」「发班级群」vs 「公司」「上班」）
- 学习状态：从评论密度和时间分布推断（寒暑假高峰=学生）
- 学业方向：从关注的UP主类型和评论内容推断文理科
- 知识面：从争论话题推断知识广度

#### C. 兴趣爱好
- 从评论指向的视频类型和内容推断
- 每个兴趣单独一项，含置信度
- 关注评论用语的「圈内人」特征

#### D. 性格特征
- 网络攻击性：吵架频率和语言风格
- 理性程度：是否引用数据、是否承认错误
- 自我认知：是否有自嘲、反思性语言
- 价值观：在争议话题上的立场

#### E. 总体画像
- 一段话概括这个人的完整形象
- 风格要刻薄但有趣

### 第四步：写锐评（核心！）

每条推断必须附带一段锐评（讽刺评论）。写作规范：

1. **句式结构**："先陈述事实 → 数据佐证 → 讽刺结论"
2. **善用类比**：把她和一个众所周知的事物类比
3. **数据做武器**：用评论数量、时间分布来嘲讽
4. **反差制造笑点**："理论上的 X，实践中的 Y"
5. **自相矛盾最佳**：找出她言行不一的地方

锐评风格参照：
- 半佛仙人：「资本韭菜」系列
- 王左中右：拆字式讽刺
- 避免：人身攻击、脏话、过度恶意

### 第五步：生成 Excel

使用 `openpyxl` 生成三个 Sheet 的 Excel 文件。

## Excel 输出规范

### 配色方案
```
主标题：白字 #FFFFFF + 深红底 #8B0000
表头：白字 + 次级红底 #B22222
分类标题：深红字 #8B0000 + 肉色底 #FFE4E1
锐评单元格：深红字 #8B0000 + 米黄底 #FFF8DC
普通文字：黑色 微软雅黑 10pt
```

### Sheet 1: 个人信息分析
- 大标题：`B站用户评论个人信息分析报告（锐评版）⚠️ 玻璃心勿入 ⚠️`
- 副标题：UID / 数据来源 / 评论数 / 生成时间 / 免责声明
- 活跃时间概况（最早/最晚时间、跨度、总数）
- 月度评论活跃趋势表
- 按类别依次输出分析项目

### Sheet 2: 关键评论原文索引
- 收集所有 `依据行号` 引用的行
- 按行号排序
- 列出分类/行号/发布时间/评论内容/视频标题

### Sheet 3: 评论时间分析
- 含 2023-2025 中国法定节假日 + 调休真实数据
- 六大板块：总览/工作日时段/寒暑假/月度明细/全天时段/终极锐评
- 每项数据都附带锐评

## 时间分析工具函数

```python
def _classify_day_full(dt):
    """完整分类：法定假日+调休+寒暑假+周末
    返回: (详细分类, 大类, 是否真工作日, 假日名称)
    """
    date_str = dt.strftime("%Y-%m-%d")
    wd = dt.weekday()
    htag = HOLIDAYS.get(date_str, None)

    if htag and '调休' not in htag:
        return (f"法定假日({htag})", "法定假日", False, htag)
    elif htag and '调休' in htag:
        return ("调休上班日(周末上班)", "工作日（含调休）", True, None)
    elif _is_school_break(dt):
        return ("寒暑假", "寒暑假", False, None)
    elif wd in (5, 6):
        return ("周末", "周末", False, None)
    else:
        return ("工作日", "工作日（含调休）", True, None)
```

## 输出文件

- `output/个人信息分析_锐评版_{timestamp}.xlsx`

## 注意事项

1. 依据行号必须指向 CSV 真实行号
2. 置信度诚实：直接证据→高，推理→中，猜测→低
3. 包含免责声明：「本分析仅供娱乐，认真你就输了」
4. 不要编造不存在的信息
5. 锐评再刻薄也不能人身攻击
6. 节假日数据需要每年更新（国务院每年发布放假安排）
