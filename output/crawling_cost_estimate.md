## 二、B站 API 限制分析

### 2.1 评论接口

```
GET https://api.bilibili.com/x/v2/reply
参数: type=1, oid={视频aid}, pn={页码}, ps={每页20}
返回: 每页最多 20 条一级评论，子回复需额外请求
```

| 限制项 | 数值 | 说明 |
|--------|------|------|
| 单次请求评论数 | 20条（一级） | 子回复需单独翻页，每页也是20条 |
| 单视频最大可获取 | ~10,000条 | 约500页，API硬限制 |
| 未登录限频 | ~3-5 req/s | 实测值，可能波动 |
| 已登录限频 | ~10-15 req/s | 需要 `SESSDATA` cookie |
| 412 触发阈值 | 约 200 req/5min | 触发 Cloudflare 验证 |
| 每日单个 IP 上限 | ~10万-50万 req | 触发 IP 封禁 |

### 2.2 实际的瓶颈

**不是你的代码效率，是 B站的反爬机制。** 主要挡路的有：

1. **频率限制（412）**：连续请求约 200 次后触发，需要等待 2-5 分钟
2. **IP 封禁**：单日请求超过阈值直接 ban IP
3. **Wbi 签名**：2023 年起部分接口需要动态签名
4. **子回复陷阱**：一级评论的子回复需要额外请求，实际请求量可能是评论数的 2-3 倍
5. **视频发现**：你得先知道有哪些视频——这本身就是一个爬取任务


## 四、硬件配置

### 4.1 推荐配置

| 组件 | 最低 | 推荐 | 说明 |
|------|------|------|------|
| CPU | 4 核 | 8 核+ | 爬虫本身不吃 CPU，但数据清洗/入库需要 |
| 内存 | 16GB | 32GB | 解析大 JSON 时峰值占用高 |
| 存储 | 2TB SSD | 4TB NVMe | 数据库写入速度关键 |
| 网络 | 100Mbps | 500Mbps+ | 带宽不是瓶颈，延迟才是 |
| 系统 | Linux | Ubuntu/Debian | 长时运行稳定性 |

### 4.2 为什么存储是最大问题

```
单条评论 JSON:          ~500 bytes
30亿条评论 JSON:         ~1.5TB
PostgreSQL 入库（含索引）: ~3TB (JSON × 2 倍膨胀)
全文搜索索引（可选）:     +1TB (Elasticsearch)
──────────────────────────────
总计:                     ~4-5TB 可用空间
```

如果要做**全文检索**（真正意义上的「搜评论数据库」），必须上 Elasticsearch，存储翻倍。

## 六、务实的建议

### 不要试图爬「全 B站」

那是大厂才做的事。B站自己的搜索团队都做不到实时索引全部评论。你应该：

1. **先跑方案 A**：选 5 万个热门视频，看看 2500 万条评论长什么样。一台家用电脑，挂一周就行。

2. **做定向爬取**：按分区（游戏/科技/生活）、按热度（>10万播放）、按时间（最近 7 天），想清楚你到底要分析什么。

3. **用已有的聚合平台做补充**：aicu.cc 已经有用户维度的历史数据。视频维度的评论可以用 `api.bilibili.com/x/v2/reply` 直接按视频拉。

4. **如果真要做大型数据库**：别单干。世界上已经有好几个团队做过 B站全站数据存档（如 Social Media Archive 项目），先看看有没有现成的。

### 最快的 MVP 方案

```python
# 不爬全站，只爬"你关心的内容"
# 目标：过去一年，科技区 + 游戏区，播放量 > 10万
# 预估：~5万个视频，~2000万条评论

时间：1-2周（单机 + 代理池）
存储：~100GB（SQLite/PostgreSQL）
成本：¥500-2000（代理 + 电费）
```

这个量级的数据已经足够做绝大部分分析了——用户画像、舆情分析、热点追踪。**不要为了「全」而牺牲「能用」。**

---

*结论：想爬全站一年评论来建库 = 1.5 年时间 + ¥6.6万经费起步。想快速出活 = 选好细分领域 + 两周 + ¥500。建议先跑 MVP，再决定要不要加码。*
